# Security Policy

## Reporting a Vulnerability

If you believe you have found a security-related issue in this project, please report it privately using GitHub's security reporting feature.

Please do not open a public issue for potential vulnerabilities.

Include as much detail as possible:

- A clear description of the issue
- Steps to reproduce
- Affected version / commit
- Potential impact
- Suggested fix (optional)

## Supported Versions

Security fixes, if provided, are generally made against the latest version of the project.

## Response Policy

This is an independent personal open-source project maintained in spare time.  
Responses and fixes are provided on a best-effort basis, and timelines cannot be guaranteed.

Thank you for helping improve the project responsibly.
